Modify src/hooks/useSpellCheck.ts for background execution (per D-PERF-03 from CONTEXT):

1. **Replace direct `localCheckSpelling(text)` call with requestIdleCallback:**
   In the `checkSpelling` function, instead of calling `localCheckSpelling(text)` synchronously:
   ```
   requestIdleCallback(() => {
     const errors = localCheckSpelling(text);
     // ... process errors
   }, { timeout: 3000 }); // 3s max wait
   ```
   Add a fallback for environments without requestIdleCallback: `const scheduleIdle = window.requestIdleCallback || ((cb: () => void) => setTimeout(cb, 1));`

2. **Cancel pending idle callbacks on new check:**
   Store the idle callback ID in a ref. Before scheduling a new one, cancel the old with `cancelIdleCallback(idRef.current)`.

3. **Wire updated scheduleSpellCheck in NoteComponent:**
   In src/components/note/index.tsx, ensure `handleChange` calls `scheduleSpellCheck(value)` on every text change (not just when `showSpellingErrors` is true). The auto-invalidation from Plan 01 handles immediate error removal; the debounced re-check catches new errors.

   Change this in handleChange:
   ```
   // OLD: if (showSpellingErrors && value !== prevValue) { scheduleSpellCheck(value, 2000); }
   // NEW: always schedule when in Bangla mode
   if (isBanglaMode) { scheduleSpellCheck(value, 2000); }
   ```

4. **Remove excessive console.log statements** from useSpellCheck.ts (keep only error logs). The hook currently has ~15 console.log calls that slow execution.
  </action>
