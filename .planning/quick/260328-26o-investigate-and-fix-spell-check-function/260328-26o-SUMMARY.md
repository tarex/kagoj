# Quick Task 260328-26o: Investigate and Fix Spell-Check Functionality

## Summary

The spell-check system works end-to-end but had several bugs causing suboptimal UX. Five issues fixed:

## Changes

### 1. Fixed underline flash on re-check (`useSpellCheck.ts`)
- **Bug:** `setSpellingErrors([])` was called before the idle callback, causing all underlines to disappear and reappear on every re-check
- **Fix:** Removed the premature clear — errors now persist until new results arrive

### 2. Clear spell errors on note switch (`note/index.tsx`)
- **Bug:** `clearSpellCheck` was never called when switching notes — stale errors from the previous note would briefly appear on the new note
- **Fix:** Added `useEffect` watching `selectedNoteIndex` that calls `clearSpellCheck`

### 3. Removed redundant ignore call (`useSpellCheck.ts`)
- **Bug:** `handleIgnoreSpelling` called both `adaptiveDictionary.learnWord()` and `addToCustomDictionary()` — they do the same thing
- **Fix:** Removed the redundant `addToCustomDictionary` call and its import

### 4. Aligned length guard (`useSpellCheck.ts`)
- **Bug:** `scheduleSpellCheck` used `> 3` but `checkSpelling` used `< 3` — texts of exactly 3 characters were handled inconsistently
- **Fix:** Changed to `>= 3` to match `checkSpelling`'s guard

### 5. Fixed cleanWord regex (`local-spell-checker.ts`)
- **Bug:** `cleanWord` regex missing `g` flag — only stripped the first punctuation character
- **Fix:** Added `g` flag to the regex

### 6. Removed unused dependency (`note/index.tsx`)
- Removed `selectedNoteIndex` from `handleChange` dependency array (was listed but never used inside the callback)

## Files Modified
- `src/hooks/useSpellCheck.ts`
- `src/components/note/index.tsx`
- `src/lib/local-spell-checker.ts`
